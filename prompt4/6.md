# 6. 투두 생성 AI

> API: `POST /todos/generate`  
> 목적: 최종 계획 기반으로 today, month, semester 단위의 투두를 생성한다.

## 6.1 System Prompt

```txt
너는 대학생의 학기별 계획을 오늘/이번 달/이번 학기 할 일로 쪼개는 실행 코치입니다.
사용자의 활성 계획과 요청 scope를 바탕으로 투두를 생성합니다.

규칙:
- 투두는 바로 실행 가능한 작은 단위로 작성합니다.
- 너무 추상적인 말은 쓰지 않습니다.
- scope는 today, month, semester 중 하나입니다.
- priority는 low, medium, high 중 하나입니다.
- dueDate는 현재 날짜와 scope를 고려해 현실적으로 정합니다.
- 각 투두는 가능한 경우 relatedPlanItemId와 연결합니다.
- title은 짧은 액션 문장으로 씁니다.
- description은 사용자가 무엇을 하면 되는지 한 문장으로 씁니다.
```

## 6.2 User Message

```txt
[현재 날짜]
{currentDate}

[요청 scope]
{scopes}

[활성 계획]
{activePlan}

[계획 항목]
{planItems}

위 계획을 바탕으로 요청 scope에 맞는 투두를 생성해줘.
```

## 6.3 호출 설정

```json
{
  "model": "gemini",
  "temperature": 0.3,
  "output_config": {
    "format": {
      "type": "json_schema",
      "schema": {
        "type": "object",
        "properties": {
          "generatedCount": { "type": "number" },
          "todos": {
            "type": "array",
            "items": {
              "type": "object",
              "properties": {
                "title": { "type": "string" },
                "description": { "type": "string" },
                "scope": { "type": "string" },
                "status": { "type": "string" },
                "priority": { "type": "string" },
                "dueDate": { "type": "string" },
                "relatedPlanItemId": { "type": "string" }
              },
              "required": [
                "title",
                "description",
                "scope",
                "status",
                "priority",
                "dueDate",
                "relatedPlanItemId"
              ],
              "additionalProperties": false
            }
          }
        },
        "required": ["generatedCount", "todos"],
        "additionalProperties": false
      }
    }
  }
}
```
