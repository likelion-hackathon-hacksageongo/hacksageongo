# 7. AI 제안 생성 규칙

> API: `/chat/sessions/{sessionId}/messages` 또는 별도 내부 로직에서 proposal 생성 시 사용한다.

## 7.1 Proposal Item Type

```txt
add_activity_candidate
update_activity_candidate
exclude_activity_candidate
add_plan_item
update_plan_item
delete_plan_item
add_todo
update_todo
delete_todo
```

## 7.2 Proposal 작성 원칙

```txt
AI가 제안한 변경사항은 바로 반영하지 않습니다.
반드시 proposal.status = "pending"으로 반환합니다.
사용자가 “AI 채팅 내용 반영하기” 버튼을 눌렀을 때만 실제 데이터에 반영합니다.

proposal.summary는 사용자가 이해할 수 있는 한 문장으로 작성합니다.
proposal.items는 변경 단위를 작게 나눕니다.
```

## 7.3 기대 출력 예시

```json
{
  "status": "pending",
  "summary": "사용자의 해외 경험 관심을 반영해 활동 후보 1개를 추가 제안합니다.",
  "items": [
    {
      "type": "add_activity_candidate",
      "payload": {
        "title": "해외 인턴",
        "category": "career_elective",
        "description": "졸업 전 해외 실무 경험을 쌓는 활동입니다.",
        "reason": "사용자가 해외 경험과 실무 경험을 모두 원한다고 말했습니다.",
        "priority": "high",
        "difficulty": "medium",
        "estimatedDuration": "2~3개월",
        "recommendedTiming": ["summer"],
        "status": "candidate"
      }
    }
  ]
}
```
